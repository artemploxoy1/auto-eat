# Плагин Auto-Eat 🍖

## Описание плагина
`Auto-Eat` — это плагин автоматического питания для ботов на панели BlockMine. Он отслеживает уровень голода бота и своевременно восстанавливает его сытость, предотвращая голодание и потерю здоровья.

Плагин поддерживает два режима работы:
1. **Обычная еда (предмет)** — бот сканирует свой инвентарь, выбирает наиболее питательную еду (которая восстанавливает больше всего единиц сытости в соответствии со встроенным справочником), берет её в активную руку и съедает.
2. **Команда сервера** — бот отправляет в чат заданную команду восстановления сытости (например, `/feed`), что полезно на серверах с донат-возможностями или специальными механиками.

## Настройки в панели управления

Параметры плагина настраиваются через графический интерфейс в настройках бота:

*   **Включить авто-еду (`autoEatEnabled`)**: Глобальный переключатель работы плагина.
*   **Способ питания (`actionsPreset`)**: Выпадающий список выбора режима работы. Доступные варианты:
    *   `Обычная еда (предмет)` (внутреннее значение: `item`)
    *   `Команда сервера` (внутреннее значение: `custom`)
*   **Команда для питания (`enableEatCommand`)**: Поле ввода команды (по умолчанию `/feed`). Благодаря встроенной во фронтенд системе условной видимости, это поле автоматически скрывается, если выбран способ питания предметами, и появляется только при переключении на команду сервера.
*   **Минимальный порог голода (`minHunger`)**: Числовой показатель сытости от 0 до 20 (по умолчанию `14`), при снижении сытости до которого запускается процесс питания.

## Интеграция с визуальным редактором (Event Graph)

Плагин генерирует два события в реальном времени, которые вы можете обрабатывать в редакторе сценариев:

### 1. `autoEat:start`
Срабатывает в момент начала приёма пищи. Передаёт в контекст графа следующие данные:
*   `username` (String) — никнейм бота.
*   `currentHunger` (Number) — уровень сытости бота на момент старта.
*   `method` (String) — выбранный способ питания (`item` или `custom`).

### 2. `autoEat:stop`
Срабатывает сразу после завершения приёма пищи. Передаёт в контекст графа следующие данные:
*   `username` (String) — никнейм бота.
*   `currentHunger` (Number) — уровень сытости бота после завершения.
